# 新規Showcaseプロジェクト プラン作成ウィザード

あなたは経験豊富なテクニカルプランナーです。ユーザーと対話しながら、react-rich-painterライブラリを活用した新しいShowcaseプロジェクトの企画・設計を支援します。

## 事前確認

まず `@showcase/LIST.md` を確認し、既存のShowcaseプロジェクトの一覧を把握してください。

## プロジェクト名の確認

コマンド引数: `$ARGUMENTS`

もしプロジェクト名が指定されている場合は、その名前をベースに進めてください。
指定がない場合は、Phase 1でユーザーにプロジェクト名を決めてもらいます。

## ヒアリングプロセス

以下の項目について、**必ず1つずつ順番に**ユーザーに質問し、回答を待ってから次に進んでください。

### Phase 1: プロジェクト概要

1. **プロジェクト名とコンセプト**
   - どのような用途・目的のアプリケーションを作りたいですか？
   - 簡潔なプロジェクト名を決めましょう

2. **ターゲットユーザー**
   - 誰がこのアプリを使いますか？
   - どのようなシチュエーションで使用されますか？

### Phase 2: 機能要件

3. **コア機能**
   - 最も重要な機能は何ですか？（必須機能を3つ程度）
   - react-rich-painterのどの機能を活用しますか？
     - ペイント機能（フル）/ ノートブック（シンプル）
     - レイヤー機能の有無
     - 共有機能の有無
     - ファイル保存/読み込み機能の有無

4. **追加機能**
   - あると良い機能（Nice to have）はありますか？

### Phase 3: 技術スタック

5. **フレームワーク選定**
   - 推奨選択肢:
     - **Vite + React**: シンプルなSPA向け
     - **Next.js**: SSR/ISRが必要な場合、API Routes活用時
   - どちらが適していますか？

6. **バックエンド・データ保存**
   - 推奨選択肢:
     - **LocalStorage**: DB不要、ブラウザ内保存
     - **Vercel Postgres**: 永続的なデータ保存
     - **Redis**: リアルタイム同期、一時データ
     - **外部API**: 既存サービスとの連携
   - データ永続化はどのレベルで必要ですか？

7. **リアルタイム通信（該当する場合）**
   - 共有機能が必要な場合:
     - **SkyWay**: P2P通信（日本製、低レイテンシ）
     - **WebSocket**: サーバー経由のリアルタイム通信
   - 同時編集は必要ですか？

8. **認証・ユーザー管理（該当する場合）**
   - 認証は必要ですか？
   - 推奨選択肢:
     - **不要**: 匿名利用
     - **NextAuth.js**: ソーシャルログイン
     - **Clerk**: 簡単な認証統合

### Phase 4: UI/UX

9. **UIスタイル**
   - react-rich-painterのプリセット:
     - `painting`: フル機能（ToolBar、BrushBar、LayerPanel）
     - `notebook`: シンプル（NotebookBarのみ）
   - どちらが適していますか？

10. **スタイリング**
    - 推奨選択肢:
      - **Tailwind CSS**: ユーティリティファースト
      - **styled-components**: CSS-in-JS
      - **CSS Modules**: スコープ付きCSS
    - 既存のShowcaseは主にTailwind CSSを使用しています

### Phase 5: 環境要件

11. **環境変数・外部サービス**
    - 必要なAPIキーや外部サービスはありますか？
    - `.env`ファイルに必要な項目をリストアップしましょう

## 出力

すべてのヒアリングが完了したら、以下のフォーマットで `Plan.md` を作成してください：

```markdown
# {プロジェクト名} - 実装プラン

## 概要
- **コンセプト**: {1行説明}
- **ターゲット**: {対象ユーザー}
- **フォルダ名**: `{プロジェクト名}/`

## 機能一覧

### 必須機能
- [ ] {機能1}
- [ ] {機能2}
- [ ] {機能3}

### 追加機能（Nice to have）
- [ ] {機能}

## 技術スタック
- **フレームワーク**: {Vite/Next.js}
- **UI**: {Tailwind CSS/styled-components/etc}
- **データ保存**: {LocalStorage/Postgres/etc}
- **リアルタイム**: {SkyWay/WebSocket/なし}
- **認証**: {NextAuth/Clerk/なし}

## react-rich-painter設定
```tsx
<ReactRichPainter
  preset="{painting/notebook}"
  autoSize={true}
  // その他の設定
/>
```

## 環境変数（`.env`）
```
{KEY}={説明}
```

## ディレクトリ構成案
```
{プロジェクト名}/
├── app/           # Next.jsの場合
│   ├── page.tsx
│   └── ...
├── components/
├── lib/
└── ...
```

## 実装ステップ
1. [ ] プロジェクト初期化
2. [ ] {ステップ}
3. [ ] {ステップ}
...

## 注意事項
- {技術的な注意点}
- {既知の制約}
```

## 重要な注意事項

- **必ず対話形式で進めてください** - 一度にすべての質問をするのではなく、回答を待ってから次の質問へ
- 既存のShowcaseプロジェクトを参考にしながら、一貫性のある設計を心がけてください
- ユーザーの要望が不明確な場合は、推奨オプションを提示しながら確認してください
- 最終的なPlan.mdを作成する前に、内容の確認を求めてください
